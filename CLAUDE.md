# zerocrew — AI 사업계획서 공동 작성 템플릿

## 프로젝트 개요
- AI 에이전트 팀을 활용한 사업계획서 공동 작성 도구
- **대상**: 모든 창업 지원사업 (예비창업동아리, 창업경진대회 등)
- **설정**: `/프로젝트설정`으로 지원사업별 규정을 `project.yaml`에 맞춤 설정

## 문서 제약 조건
project.yaml 참조 (분량, 서체, 마감, 제출처, 제출 서류)

## 예산 편성
상세 예산은 `project.yaml`의 budget 섹션 참조

## 섹션 구조 & 페이지 배분
섹션 구조는 `project.yaml`의 document.sections 참조

## 시작점: project.yaml + plan.md
- `project.yaml`(규정)과 `plan.md`(아이디어)가 모든 작업의 입력
- 모든 에이전트는 작업 전 반드시 두 파일을 먼저 읽음
- plan.md의 [빠른 시작] 섹션이 비어있으면 에이전트가 작업을 거부하고 작성을 안내함
- plan.md 작성 예시는 plan.example.md, project.yaml 작성 예시는 project.example.yaml 참고

## 디렉터리 구조
```
project.yaml               ← 규정: 예산, 포맷, 섹션, 심사 기준
plan.md                    ← 아이디어: 아이템 정보, 팀 정보
docs/                      ← 공고문, 양식 등 참고 문서
versions/
  A-1/drafts/00~04.md      ← A팀원 1차 버전
  A-2/drafts/00~04.md      ← A팀원 2차 버전
  A-3/drafts/00~04.md      ← A팀원 3차 버전
  B-1/ ... B-3/            ← B팀원 3개 버전
  C-1/ ... C-3/            ← C팀원 3개 버전
  D-1/ ... D-3/            ← D팀원 3개 버전
  A/drafts/00~04.md        ← A팀원이 선택한 최종 버전
  B/drafts/00~04.md        ← B팀원이 선택한 최종 버전
  C/drafts/00~04.md        ← C팀원이 선택한 최종 버전
  D/drafts/00~04.md        ← D팀원이 선택한 최종 버전
drafts/                    ← 버전심사관이 선정한 최종 섹션 조합
final/                     ← 문서통합자가 통합한 최종본
research/                  ← 리서치 산출물
```

## 워크플로우
0. **계획 작성** — `/계획작성`으로 대화형 인터뷰 또는 직접 `plan.md` 편집 (전원 함께)
1. **각자 3버전 작성** — 각 팀원이 `/초안작성 X-1`, `/초안작성 X-2`, `/초안작성 X-3`으로 3개 버전 생성
2. **각자 베스트 선택** — 3개 중 스토리가 가장 자연스러운 1개를 `versions/X/drafts/`에 선택
3. **비교 선정** — 버전심사관 에이전트가 4명의 베스트 버전을 스토리 중심으로 비교, 최종 조합 선정
4. **최종 통합** — 문서통합자 에이전트가 선정된 조합을 `final/사업계획서.md`로 통합
5. **검토** — 품질검토관 + 악마의변호인으로 품질/일관성/약점 체크
6. **변환** — 한글변환기 에이전트로 최종 hwpx 파일 생성

## 컨벤션
- 파일명: 한글 허용, 섹션은 00~04 번호 프리픽스
- 리서치: `research/` 하위에 출처 URL 필수 명시
- 커밋 메시지: 한글, `[섹션명] 내용 요약` 형식
