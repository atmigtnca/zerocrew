## 사업화 전략 로드맵

JClaw의 로드맵은 "빠르게 만들고, 빠르게 검증하고, 빠르게 확산한다"는 원칙으로 설계했다. 교내 인프라가 이미 존재하므로, 서버 구축 단계를 건너뛰고 바로 서비스 개발에 집중할 수 있다.

| 단계 | 기간 | 핵심 활동 | 산출물 | 비용 | KPI |
|------|------|----------|--------|------|-----|
| **1. 설계** | 5~6월 | 사용자 인터뷰 20건(비전공 10 + 전공 10), 와이어프레임, 시스템 아키텍처, Mattermost Bot 프로토타입 | 기획서, 와이어프레임, 아키텍처 문서 | 50만원 | 인터뷰 20건 완료 |
| **2. MVP** | 7~9월 | React 웹 빌더 UI, 백엔드 API(에이전트 CRUD), LLM API 연동, Mattermost Bot 자동 등록, OpenStack 배포 | 동작하는 MVP | 170만원 | 3대 핵심 기능 구현 |
| **3. 베타** | 10~11월 | 베타 테스터 100명 운영, 버그 수정, UX 개선, 학과별 킬러 에이전트 사전 제작 | 안정화 버전, 피드백 보고서 | 50만원 | 테스터 100명, 에이전트 50개 |

### 차별화: 우리만 할 수 있는 3가지

1. **교내 인프라 네이티브**: JClaw는 전북대 OpenStack 위에서 직접 실행된다. 외부 클라우드를 경유하지 않으므로 지연시간이 최소이고, 네트워크 비용이 0이다. Dify나 OpenClaw를 교내에 배포하려면 별도 서버 구축이 필요하지만, JClaw는 이미 있는 인프라에 최적화되어 있다.

2. **Mattermost 퍼스트**: 대부분의 AI 빌더는 범용 API 출력을 제공한다. JClaw는 처음부터 Mattermost를 1순위 배포 채널로 설계한다. 에이전트 생성 → Mattermost Bot 등록이 하나의 플로우다. 전북대 학생이 매일 쓰는 메신저에서 바로 AI 에이전트를 만나게 된다.

3. **LLM 자원의 민주화**: 교내 LLM API는 유한하다. JClaw는 사용자별 할당량, 실시간 모니터링, 경량 모델 자동 전환 기능으로 제한된 자원을 공정하게 배분한다. "소수가 독점하는 API"가 아니라 "모두가 적정하게 쓰는 API"를 실현한다.

### 예산 효율성

300만원의 지원금은 인프라 비용 0원이라는 조건 덕분에 **전액 서비스 품질(UI 디자인, 브랜딩)과 운영(회의, 테스트)에 집중 투입**된다. 일반적으로 초기 서비스의 클라우드 비용이 월 30~50만원인 것을 감안하면, 교내 인프라 활용은 8개월 기준 240~400만원의 비용 절감 효과가 있다.
