# 팀 구성

## 창업자 역량

JClaw 팀은 "만들 수 있는 사람들이 만들어야 할 것을 만든다"는 원칙으로 구성되었다. 컴퓨터공학부 4명 전원이 교내 OpenStack 클라우드를 직접 사용해본 경험이 있으며, Mattermost를 일상적으로 활용하는 실사용자이기도 하다. 즉, 우리는 JClaw의 개발자인 동시에 첫 번째 사용자다.

4명의 역할은 JClaw의 기술 아키텍처와 정확히 매핑된다: 웹 프론트엔드(에이전트 빌더 UI), 백엔드 API(에이전트 실행 엔진), 인프라(OpenStack 배포 및 운영), AI/LLM 연동(모델 통합 및 Mattermost Bot). 각 역할이 독립적 모듈이므로 병렬 개발이 가능하고, 한 명이 빠져도 나머지가 커버할 수 있는 구조다.

## 시너지 효과

JClaw의 핵심은 "비전공자 문제를 전공자 기술로 푸는 것"이다. 4명 모두 컴퓨터공학 전공이므로 기술적 의사소통에 오해가 없고, Git 기반 협업, 코드 리뷰, 스프린트 방식의 개발이 자연스럽다.

특히 팀 전원이 전북대 Mattermost와 OpenStack의 실사용자라는 점이 결정적이다. 교내 시스템의 제약 조건(API 한계, 인증 체계, 네트워크 환경)을 체감하고 있어, 외부 개발팀이 놓치기 쉬운 교내 특수 상황에 최적화된 설계가 가능하다.

**역할별 독립 모듈 구조**로, 개발 진행 중 한 모듈이 지연되더라도 다른 모듈은 독립적으로 진행 가능하다. 주 1회 통합 미팅에서 모듈 간 인터페이스만 맞추면, 최종 통합 시 충돌을 최소화할 수 있다.
