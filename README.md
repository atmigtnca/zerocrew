# zerocrew

zeroone 팀의 **zerothon** — AI 도구를 활용한 2시간 사업계획서 스프린트

---

## 퀵스타트

```bash
git clone <repo-url> && cd zerocrew
pip install hwpx-mcp-server    # 최종 hwpx 변환용
claude                          # Claude Code 실행
```

---

## 어떻게 동작하나요?

```
plan.md (아이디어)
    ↓
4명이 각자 전체 사업계획서 작성 (AI 에이전트 지원)
    ↓
plan-selector가 4개 버전을 스토리 중심으로 비교
    ↓
integrator가 하나의 문서로 통합
    ↓
hwpx-converter가 제출용 한글 파일 생성
```

---

## 2시간 타임라인

| 시간 | 단계 | 누가 | 무엇을 |
|------|------|------|--------|
| **0~10분** | 브리프 | 전원 | `plan.md`에 아이디어, 팀 정보 채우기 |
| **10~40분** | 리서치 | 각자 | AI에게 시장조사/경쟁분석 요청 |
| **40~80분** | 초안 작성 | 각자 | `versions/X/drafts/`에 전체 5개 섹션 작성 |
| **80~100분** | 비교 선정 | PM | 4개 버전 스토리 비교 → 베스트 조합 |
| **100~110분** | 통합 | PM | 선정된 조합 → `final/사업계획서.md` |
| **110~115분** | 검토 | PM | 필수항목/일관성 최종 체크 |
| **115~120분** | 변환 | PM | `final/사업계획서.hwpx` 생성 |

## 사용법

### STEP 0. 브리프 작성 (전원, 0~10분)

`plan.md`를 열고 함께 채웁니다. 이 파일이 모든 AI 작업의 입력입니다.

```
필수 항목:
- 아이템명, 한 줄 소개
- 해결하는 문제, 타겟 사용자
- 핵심 기능 2~3개
- 팀원 정보 (이름, 학과, 역할)
```

### STEP 1. 각자 전체 사업계획서 작성 (각자, 10~80분)

각 팀원은 자기 폴더(`versions/A~D/drafts/`)에서 **전체 5개 섹션**을 작성합니다.

**Claude에게 이렇게 말하세요:**

한 번에 전체 작성:
```
"plan.md 읽고 versions/A/drafts/ 에 전체 사업계획서 작성해줘"
```

또는 섹션별로 나눠서:
```
"plan.md 기반으로 versions/A/drafts/00-아이템개요.md 작성해줘"
"시장조사해줘"                    ← market-researcher 에이전트 동작
"경쟁사 분석해줘"                  ← strategy-advisor 에이전트 동작
"versions/A/drafts/02-문제인식.md 작성해줘"
"예산 검증해줘"                    ← financial-analyst 에이전트 동작
...
```

> 같은 brief에서 출발하지만, 각자의 AI가 다른 데이터와 관점을 제공합니다.

### STEP 2. 비교 → 통합 → 변환 (PM, 80~120분)

```
"versions/ 의 4개 버전 비교해서 최종 조합 만들어줘"
```
→ plan-selector가 **스토리 일관성** 기준으로 베스트 조합 선정
→ "문제 → 해결 → 시장 → 전략 → 예산"이 하나의 이야기로 이어지는 조합을 찾음

```
"최종 통합해줘"
```
→ integrator가 선정된 조합을 톤/용어 통일하여 `final/사업계획서.md` 생성

```
"검토해줘"
```
→ document-reviewer가 필수항목, 수치 일관성, 분량 체크

```
"hwpx로 변환해줘"
```
→ hwpx-converter가 `final/사업계획서.hwpx` 생성 (중고딕 12pt, 줄간격 160%)

## AI 에이전트

작업 중 자연스럽게 동작하는 전문 에이전트들입니다.

### 작성 지원 (각자 작성 단계에서 사용)

| 에이전트 | 역할 | 이렇게 부르세요 |
|---------|------|----------------|
| **market-researcher** | 시장 규모(TAM/SAM/SOM), 트렌드, 통계 수집 | "시장조사해줘", "시장 규모 알려줘" |
| **strategy-advisor** | 경쟁사 분석, 차별화 전략, 사업화 로드맵 | "경쟁사 분석해줘", "로드맵 짜줘" |
| **financial-analyst** | 예산 검증, 집행 규정 체크, 자금조달 계획 | "예산 검증해줘", "사업비 체크해줘" |

### 통합/검토 (PM이 마지막 단계에서 사용)

| 에이전트 | 역할 | 이렇게 부르세요 |
|---------|------|----------------|
| **plan-selector** | 4개 버전을 스토리 중심으로 비교, 베스트 조합 선정 | "버전 비교해줘", "베스트 조합 만들어줘" |
| **document-reviewer** | 필수항목 체크, 수치 일관성, 분량, 심사 관점 피드백 | "검토해줘", "피드백 줘" |
| **integrator** | 섹션 통합, 톤/문체/용어 통일, 분량 조정 | "통합해줘", "하나로 합쳐줘" |
| **hwpx-converter** | 마크다운 → hwpx 한글 파일 변환 | "hwpx로 변환해줘" |

## 스킬 (자동 트리거)

관련 작업 시 자동으로 프레임워크와 가이드라인을 제공합니다.

| 스킬 | 제공하는 것 |
|------|-----------|
| **market-research** | TAM/SAM/SOM 추정 공식, 국내 통계 출처 목록 (통계청, KOSIS 등) |
| **competitive-analysis** | 경쟁사 비교 테이블 템플릿, 포지셔닝 맵 설계 가이드 |
| **financial-planning** | 300만원 집행 규정, 항목별 한도, 금지 항목, 정산 주의사항 |
| **document-quality** | 필수항목 체크리스트, 섹션별 분량 기준, 심사 배점 |
| **hwpx-export** | 마크다운 → hwpx 변환 절차, MCP 도구 매핑 |
