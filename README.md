# zerocrew

AI 에이전트 팀과 함께하는 사업계획서 공동 작성 도구

> 어떤 창업 지원사업이든 OK — `/프로젝트설정`으로 규정을 맞추고, `plan.md`에 아이디어만 채우면 시작!

---

## 처음이신가요? 여기서 시작하세요

### 1단계: 준비물 확인

| 필요한 것 | 확인 |
|----------|------|
| Git 설치 | 터미널에서 `git --version` 입력 → 버전 나오면 OK |
| Claude Code 설치 | 터미널에서 `claude --version` 입력 → 버전 나오면 OK |
| 이 저장소 클론 | 아래 명령어 실행 |

```bash
# 저장소 클론
git clone <repo-url>
cd zerocrew

# Claude Code 실행
claude
```

> **Claude Code가 없다면?**
> ```bash
> npm install -g @anthropic-ai/claude-code
> ```
> 설치 후 `claude` 명령어로 실행하면 됩니다.

### 2단계: hwpx 설정 (처음 한 번)

```
/hwpx설정
```
AI가 자동으로 hwpx-mcp-server를 설정합니다 (uvx 확인 → .mcp.json 생성).
설정 후 Claude Code를 재시작하면 **hwpx MCP 서버 승인 팝업**이 뜹니다.
**"허용"을 눌러주세요.** (한글 파일 변환에 필요합니다)

### 3단계: 프로젝트 설정 (처음 한 번)

```
/프로젝트설정
```
AI가 질문하면 대답하세요. 지원사업 규정, 예산, 문서 규격 등을 `project.yaml`에 자동 저장합니다.
공고문이 `docs/` 폴더에 있으면 자동으로 읽어서 정보를 추출합니다.

> **처음 사용 시 반드시 `/프로젝트설정`을 먼저 실행하세요.** 지원사업마다 예산, 분량, 섹션 구조가 다릅니다.

## 어떻게 동작하나요?

### 수동 모드 (팀원 각자 작성)
```
project.yaml (규정) + plan.md (아이디어)
    ↓
4명이 각자 3버전씩 사업계획서 작성
    ↓
각자 베스트 1개 선택 (12개 → 4개)
    ↓
버전심사관이 4개를 스토리 중심으로 비교
    ↓
문서통합자가 하나의 문서로 통합
    ↓
품질검토관 + 악마의변호인이 검토
    ↓
한글변환기가 제출용 hwpx 파일 생성
```

### 자동 모드 (AI가 전부 처리)
```
project.yaml (규정) + plan.md (아이디어)
    ↓
/자동완성  ← 이 명령어 하나로 끝
    ↓
오케스트레이터가 에이전트 팀을 자율 조율
(리서치 → 초안2버전 → 비판/보강 → 비교 → 통합 → 검토루프)
    ↓
final/사업계획서.md 자동 생성
```

## 사용법 — 이것만 알면 됩니다

Claude Code 안에서 `/명령어`를 입력하면 AI가 알아서 작업합니다.
**직접 코드를 쓸 필요 없습니다.**

### 팀원용 명령어 (각자 사용)

| 명령어 | 하는 일 | 예시 |
|--------|--------|------|
| `/초안작성 A-1` | 내 1차 버전 사업계획서 자동 작성 | `/초안작성 A-1`, `/초안작성 A-2`, `/초안작성 A-3` |
| `/베스트선택 A` | 내 3개 버전 비교 후 베스트 1개 선택 | `/베스트선택 A` |
| `/시장조사` | 시장 규모, TAM/SAM/SOM, 트렌드 조사 | `/시장조사` |
| `/전략분석` | 경쟁사 분석, 차별화 전략, 로드맵 수립 | `/전략분석` |
| `/예산검토` | 예산 항목별 규정 부합 여부 체크 | `/예산검토` |
| `/비판리뷰` | 내 초안을 악마의 변호인이 비판적으로 리뷰 | `/비판리뷰` |

### PM용 명령어 (통합/완성 단계)

| 명령어 | 하는 일 | 예시 |
|--------|--------|------|
| `/버전비교` | 4명의 베스트 버전을 스토리 중심으로 비교, 최종 조합 선정 | `/버전비교` |
| `/문서통합` | 선정된 조합을 하나의 문서로 통합 | `/문서통합` |
| `/품질검토` | 필수항목, 수치 일관성, 분량 체크 | `/품질검토` |
| `/한글변환` | 최종 hwpx(한글) 파일 생성 | `/한글변환` |
| `/전체완성` | 위 과정을 순서대로 한 번에 실행 | `/전체완성` |

### 설정/자동화 명령어

| 명령어 | 하는 일 | 예시 |
|--------|--------|------|
| `/프로젝트설정` | 대화형 인터뷰로 project.yaml 자동 생성 | `/프로젝트설정` |
| `/자동완성` | plan.md만 쓰면 AI가 전체 사업계획서 자율 완성 | `/자동완성` |
| `/hwpx설정` | hwpx-mcp-server 자동 설치/설정 | `/hwpx설정` |

> **명령어 없이도 됩니다!** 그냥 한국어로 말해도 AI가 알아듣습니다:
> ```
> "시장조사해줘"
> "경쟁사 분석 좀 해줘"
> "내 초안 비판적으로 리뷰해줘"
> ```

## 단계별 상세 가이드

### STEP 0. 계획 수립 (전원)

`plan.md` 파일을 열고 **함께** 채웁니다. 이 파일이 모든 AI 작업의 출발점입니다.

```
필수로 채울 항목:
- 아이템명
- 한 줄 소개
- 해결하는 문제
- 타겟 사용자
- 핵심 기능 2~3개
- 팀원 정보 (이름, 학과, 역할)
```

> plan.md가 비어있으면 AI가 작업을 거부합니다. 반드시 먼저 채우세요.

### STEP 1. 각자 3버전 작성 (각자)

각 팀원은 **자기 알파벳**(A, B, C, D)을 정하고 3개 버전을 만듭니다.

```
/초안작성 A-1     ← 1차 버전
/초안작성 A-2     ← 2차 버전 (AI가 다른 관점으로 작성)
/초안작성 A-3     ← 3차 버전 (또 다른 내러티브)
```

각 명령어가 **3단계 프로세스**를 자동 실행합니다:

```
Phase 1: 초안 생성 ──→ 5개 섹션 파일 자동 작성 (시장조사/경쟁분석 포함)
    ↓
Phase 2: 악마의변호인 공격 ──→ 초안의 약점을 찾아냄
    ↓
Phase 3: 보강 ──→ 찾아낸 약점을 바탕으로 초안 자동 수정
```

즉, `/초안작성 A-1` 한 번이면 **이미 비판을 거쳐 보강된 버전**이 나옵니다.

```
versions/A-1/drafts/
├── 00-아이템개요.md    (요약서 1p)
├── 01-팀구성.md        (0.5p)
├── 02-문제인식.md      (2.5p) ← 시장조사 포함
├── 03-실현가능성.md    (1.5p) ← 경쟁분석 포함
└── 04-사업비편성.md    (0.5p) ← 예산검증 포함
```

> **기다리는 동안** 이전 버전을 읽어보세요. 추가로 직접 비판을 요청할 수도 있습니다:
> ```
> /비판리뷰 versions/A-1/drafts    ← 한 번 더 비판받기
> "02-문제인식 시장 데이터를 더 보강해줘"
> ```

### STEP 2. 각자 베스트 선택 (각자)

3개 버전 중 **스토리가 가장 자연스러운 1개**를 고릅니다:

```
/베스트선택 A     ← AI가 3개 비교해서 추천 + 선택한 버전을 versions/A/drafts/에 복사
```

> AI 추천을 무시하고 직접 고를 수도 있습니다. 그냥 말하세요:
> ```
> "A-2가 제일 나은 것 같아. 그걸로 선택해줘"
> ```

### STEP 3. 비교 + 통합 + 검토 (PM)

4명의 베스트가 모이면 PM이 순서대로 실행합니다:

```
/버전비교        ← 4명의 베스트 중 스토리가 가장 잘 이어지는 조합 선정
/문서통합        ← 선정된 조합을 하나의 문서로 합침
/품질검토        ← 필수항목 누락, 수치 불일치, 분량 체크
/비판리뷰        ← 악마의 변호인이 모든 약점을 공격
```

> 또는 `/전체완성` 하나로 위 과정을 한 번에 실행할 수 있습니다.

### STEP 4. 한글 변환 (PM)

```
/한글변환        ← final/사업계획서.hwpx 생성
```

생성된 hwpx 파일을 **반드시 한글 프로그램에서 열어** 확인하세요:
- [ ] 서체가 중고딕 12pt인가?
- [ ] 줄간격이 160%인가?
- [ ] 요약서 1p + 본문 5p 이내인가?
- [ ] 표가 깨지지 않았는가?

## AI 에이전트 소개

이 프로젝트에는 8명의 AI 전문가가 있습니다:

### 작성 지원팀

| 이름 | 역할 | 언제 쓰나요? |
|------|------|-------------|
| **시장분석가** | 시장 규모(TAM/SAM/SOM), 트렌드, 통계 수집 | 시장 데이터가 필요할 때 |
| **전략자문가** | 경쟁사 분석, 차별화 전략, 사업화 로드맵 | 전략 섹션 작성할 때 |
| **예산분석가** | 예산 검증, 집행 규정 체크, 자금조달 계획 | 예산 섹션 작성할 때 |

### 검토팀

| 이름 | 역할 | 언제 쓰나요? |
|------|------|-------------|
| **품질검토관** | 필수항목 체크, 수치 일관성, 분량, 심사 관점 피드백 | 초안 완성 후 |
| **악마의변호인** | 모든 주장에 반대하며 약점 찾기 (의도적으로 비판적) | 제출 전 최종 점검 |

### 통합/변환팀

| 이름 | 역할 | 언제 쓰나요? |
|------|------|-------------|
| **버전심사관** | 4개 버전을 스토리 중심으로 비교, 베스트 조합 선정 | 4명 초안 모인 후 |
| **문서통합자** | 섹션 통합, 톤/문체/용어 통일, 분량 조정 | 베스트 조합 선정 후 |
| **한글변환기** | 마크다운을 hwpx(한글) 파일로 변환 | 최종본 확정 후 |

### 설정/자동화팀

| 이름 | 역할 | 언제 쓰나요? |
|------|------|-------------|
| **설정도우미** | 대화형 인터뷰로 project.yaml 자동 생성 | 프로젝트 최초 셋업 시 |
| **오케스트레이터** | 에이전트 팀을 자율 조율하여 사업계획서 자동 완성 | `/자동완성` 실행 시 |

## 폴더 구조

```
zerocrew/
├── project.yaml                    ← 규정: 예산, 포맷, 섹션, 심사 기준
├── plan.md                         ← 아이디어: 아이템 정보, 팀 정보
│
├── docs/                            ← 공고문, 양식 등 참고 문서
│
├── versions/                        ← 팀원별 독립 버전
│   ├── A-1/, A-2/, A-3/            ← A팀원의 3개 버전
│   ├── A/drafts/                    ← A팀원이 선택한 베스트
│   ├── B-1/, B-2/, B-3/            ← B팀원의 3개 버전
│   ├── B/drafts/                    ← B팀원이 선택한 베스트
│   └── ...                          ← C, D팀원도 동일
│
├── drafts/                          ← 버전심사관이 선정한 최종 조합
│
├── final/                           ← 최종 산출물
│   ├── 사업계획서.md                 ← 문서통합자가 생성
│   └── 사업계획서.hwpx               ← 한글변환기가 생성 (제출용!)
│
├── research/                        ← 리서치 결과물
│
├── .claude/
│   ├── agents/                      ← AI 에이전트 10종
│   ├── commands/                    ← /슬래시 명령어 13종
│   └── skills/                      ← 도메인 지식 스킬 6종
│
├── CLAUDE.md                        ← AI 설정 파일
└── README.md                        ← 지금 읽고 있는 이 파일
```

## 자주 묻는 질문

### Q. Claude Code가 뭔가요?
터미널(명령줄)에서 동작하는 AI 코딩 도우미입니다. 이 프로젝트에서는 사업계획서 작성을 도와주는 AI 팀으로 활용합니다. 코딩 경험이 없어도 사용할 수 있습니다.

### Q. 터미널은 어떻게 여나요?
- **Mac**: `Cmd + Space` → "터미널" 검색 → 엔터
- **Windows**: `Win + R` → `cmd` 입력 → 엔터 (또는 Windows Terminal 사용)
- **Linux**: `Ctrl + Alt + T`

### Q. Claude Code를 실행했는데 아무것도 안 돼요
1. `zerocrew` 폴더 안에서 실행했는지 확인: `cd zerocrew && claude`
2. hwpx MCP 서버 승인 팝업이 떴으면 "허용" 클릭

### Q. `/초안작성 A` 했는데 "plan.md를 먼저 작성하세요"라고 나와요
plan.md가 비어있습니다. STEP 0을 먼저 완료하세요.

### Q. 명령어를 잘못 입력했어요
걱정하지 마세요. 그냥 한국어로 원하는 걸 말하면 됩니다:
```
"시장조사 다시 해줘"
"아까 작성한 02-문제인식.md 수정해줘"
"versions/A/drafts/ 전체 다시 작성해줘"
```

### Q. 다른 팀원이 작성한 내용을 보고 싶어요
```
"versions/B/drafts/ 전체 읽어줘"
"B팀원 버전이랑 내 버전 비교해줘"
```

### Q. 비판리뷰가 너무 혹독한데요?
의도된 것입니다. 악마의변호인은 **일부러** 최대한 비판적으로 리뷰합니다. 심사위원이 던질 질문을 미리 찾아내서, 제출 전에 약점을 보강하는 것이 목적입니다. 상처받지 마세요!

### Q. Git 사용법을 모르겠어요
이 프로젝트에서 알아야 할 Git 명령어는 딱 3개입니다:
```bash
git pull                    # 다른 팀원의 최신 변경사항 가져오기
git add . && git commit -m "내 초안 작성"   # 내 작업 저장
git push                    # 팀원에게 공유
```
