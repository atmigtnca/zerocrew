# zerocrew

AI 에이전트 팀과 함께하는 사업계획서 공동 작성 도구

> 어떤 창업 지원사업이든 OK — `/프로젝트설정`으로 규정을 맞추고, `plan.md`에 아이디어만 채우면 시작!

---

## 처음이신가요? 여기서 시작하세요

### 1단계: 준비물 확인

| 필요한 것 | 확인 |
|----------|------|
| Git 설치 | 터미널에서 `git --version` 입력 → 버전 나오면 OK |
| Claude Code 설치 | 터미널에서 `claude --version` 입력 → 버전 나오면 OK |
| 이 저장소 클론 | 아래 명령어 실행 |

```bash
# 저장소 클론
git clone <repo-url>
cd zerocrew

# Claude Code 실행
claude
```

> **Claude Code가 없다면?**
> ```bash
> npm install -g @anthropic-ai/claude-code
> ```
> 설치 후 `claude` 명령어로 실행하면 됩니다.

### 2단계: hwpx 설정 (처음 한 번)

```
/hwpx설정
```
AI가 자동으로 hwpx-mcp-server를 설정합니다 (uvx 확인 → .mcp.json 생성).
설정 후 Claude Code를 재시작하면 **hwpx MCP 서버 승인 팝업**이 뜹니다.
**"허용"을 눌러주세요.** (한글 파일 변환에 필요합니다)

### 3단계: 프로젝트 설정 (처음 한 번)

```
/프로젝트설정
```
AI가 질문하면 대답하세요. 지원사업 규정, 예산, 문서 규격 등을 `project.yaml`에 자동 저장합니다.
공고문이 `docs/` 폴더에 있으면 자동으로 읽어서 정보를 추출합니다.

> **처음 사용 시 반드시 `/프로젝트설정`을 먼저 실행하세요.** 지원사업마다 예산, 분량, 섹션 구조가 다릅니다.

## 어떻게 동작하나요?

### 수동 모드 (팀원 각자 작성)
```
project.yaml (규정) + plan.md (아이디어)
    ↓
4명이 각자 3버전씩 사업계획서 작성
    ↓
각자 베스트 1개 선택 (12개 → 4개)
    ↓
버전심사관이 4개를 스토리 중심으로 비교
    ↓
문서통합자가 하나의 문서로 통합
    ↓
품질검토관 + 악마의변호인이 검토
    ↓
한글변환기가 제출용 hwpx 파일 생성
```

### 자동 모드 (AI가 전부 처리)
```
project.yaml (규정) + plan.md (아이디어)
    ↓
/자동완성  ← 이 명령어 하나로 끝
    ↓
오케스트레이터가 에이전트 팀을 자율 조율
(리서치 → 초안2버전 → 비판/보강 → 비교 → 통합 → 검토루프)
    ↓
final/사업계획서.md 자동 생성
```

## 사용법 — 이것만 알면 됩니다

Claude Code 안에서 `/명령어`를 입력하면 AI가 알아서 작업합니다.
**직접 코드를 쓸 필요 없습니다.**

### 팀원용 명령어 (각자 사용)

| 명령어 | 하는 일 | 예시 |
|--------|--------|------|
| `/초안작성 A-1` | 내 1차 버전 사업계획서 자동 작성 | `/초안작성 A-1`, `/초안작성 A-2`, `/초안작성 A-3` |
| `/베스트선택 A` | 내 3개 버전 비교 후 베스트 1개 선택 | `/베스트선택 A` |
| `/시장조사` | 시장 규모, TAM/SAM/SOM, 트렌드 조사 | `/시장조사` |
| `/전략분석` | 경쟁사 분석, 차별화 전략, 로드맵 수립 | `/전략분석` |
| `/예산검토` | 예산 항목별 규정 부합 여부 체크 | `/예산검토` |
| `/비판리뷰` | 내 초안을 악마의 변호인이 비판적으로 리뷰 | `/비판리뷰` |

### PM용 명령어 (통합/완성 단계)

| 명령어 | 하는 일 | 예시 |
|--------|--------|------|
| `/버전비교` | 4명의 베스트 버전을 스토리 중심으로 비교, 최종 조합 선정 | `/버전비교` |
| `/문서통합` | 선정된 조합을 하나의 문서로 통합 | `/문서통합` |
| `/품질검토` | 필수항목, 수치 일관성, 분량 체크 | `/품질검토` |
| `/한글변환` | 최종 hwpx(한글) 파일 생성 | `/한글변환` |
| `/전체완성` | 위 과정을 순서대로 한 번에 실행 | `/전체완성` |

### 설정/자동화 명령어

| 명령어 | 하는 일 | 예시 |
|--------|--------|------|
| `/프로젝트설정` | 대화형 인터뷰로 project.yaml 자동 생성 | `/프로젝트설정` |
| `/계획작성` | 대화형 인터뷰로 plan.md 자동 생성 | `/계획작성` |
| `/자동완성` | plan.md만 쓰면 AI가 전체 사업계획서 자율 완성 | `/자동완성` |
| `/hwpx설정` | hwpx-mcp-server 자동 설치/설정 | `/hwpx설정` |

> **명령어 없이도 됩니다!** 그냥 한국어로 말해도 AI가 알아듣습니다:
> ```
> "시장조사해줘"
> "경쟁사 분석 좀 해줘"
> "내 초안 비판적으로 리뷰해줘"
> ```

## 작업 순서

1. **계획** — `/계획작성`으로 대화형 인터뷰 또는 직접 `plan.md` 편집
2. **작성** — 각자 3버전 생성
3. **선택** — 각자 베스트 1개 선택
4. **통합** — 비교 후 하나로 합치기
5. **검토** — 품질 체크 + 비판 리뷰
6. **변환** — 한글(hwpx) 파일 생성

> `/자동완성` 하나로 1~6을 전부 건너뛸 수도 있습니다.

## 단계별 가이드

### STEP 0. plan.md 채우기

모든 AI 작업의 출발점입니다. 두 가지 방법 중 편한 걸 선택하세요:

- **방법 1: `/계획작성`** — AI가 질문하면 대답하는 방식으로 plan.md를 자동 생성합니다. 마크다운을 몰라도 OK.
- **방법 2: 직접 편집** — `plan.md`를 vim 등으로 열어서 직접 채웁니다.

아이템명, 한 줄 소개, 해결하는 문제, 타겟 사용자, 핵심 기능, 팀원 — 이 6가지만 있으면 시작할 수 있습니다.

**상세 정보** 섹션은 선택이지만, 채울수록 AI가 더 정확한 사업계획서를 만들어냅니다.
예를 들어 "왜 이걸 만들려고 하는지"를 채우면 문제 인식 섹션의 설득력이 올라가고,
"벤치마킹 서비스"를 적으면 경쟁사 분석이 더 구체적으로 나옵니다.

> plan.md가 비어있으면 AI가 작업을 거부합니다. 반드시 먼저 채우세요.
> 작성 예시는 `plan.example.md`를 참고하세요.

### STEP 1. 각자 3버전 작성

팀원마다 알파벳(A, B, C, D)을 정하고, 각자 3개 버전을 만듭니다.

```
/초안작성 A-1     ← 1차 버전
/초안작성 A-2     ← 2차 버전 (AI가 다른 관점으로 작성)
/초안작성 A-3     ← 3차 버전 (또 다른 내러티브)
```

**한 번 실행하면 AI가 3단계를 자동으로 처리합니다:**

1. **초안 생성** — 시장조사와 경쟁분석을 포함해서 5개 섹션 파일을 작성합니다
2. **악마의변호인 공격** — 작성된 초안의 논리적 허점과 약점을 찾아냅니다
3. **보강** — 찾아낸 약점을 바탕으로 초안을 자동 수정합니다

즉 `/초안작성 A-1` 한 번이면 **이미 비판을 거쳐 보강된 버전**이 나옵니다.
결과물은 아래 경로에 저장됩니다:

```
versions/A-1/drafts/
├── 00-아이템개요.md      ← 요약서
├── 01-팀구성.md          ← 팀 소개
├── 02-문제인식.md        ← 시장조사 포함
├── 03-실현가능성.md      ← 경쟁분석 포함
└── 04-사업비편성.md      ← 예산검증 포함
```

> 기다리는 동안 이전 버전을 읽어보거나, 추가 비판을 요청할 수 있습니다:
> ```
> /비판리뷰 versions/A-1/drafts
> "02-문제인식 시장 데이터를 더 보강해줘"
> ```

### STEP 2. 베스트 선택

3개 버전 중 **스토리가 가장 자연스러운 1개**를 고릅니다.

```
/베스트선택 A
```

AI가 3개 버전을 비교 분석해서 추천해주고, 선택된 버전을 `versions/A/drafts/`로 복사합니다.
AI 추천이 마음에 안 들면 직접 고를 수도 있습니다:
```
"A-2가 제일 나은 것 같아. 그걸로 선택해줘"
```

### STEP 3. 통합 + 검토 (PM)

팀원 전원의 베스트가 모이면 PM이 순서대로 실행합니다.

```
/버전비교     ← 팀원별 베스트에서 스토리가 가장 잘 이어지는 섹션 조합을 선정
/문서통합     ← 선정된 조합을 하나의 문서로 합치고, 톤/문체/용어를 통일
/품질검토     ← 필수항목 누락, 수치 불일치, 분량 초과 등을 체크
/비판리뷰     ← 심사위원 관점에서 모든 약점을 공격하고 보강 방향을 제시
```

검토 결과 수정이 필요하면 AI에게 바로 요청하면 됩니다:
```
"02-문제인식에서 TAM 수치를 최신 데이터로 업데이트해줘"
"03-실현가능성 로드맵이 너무 추상적이야, 구체적으로 수정해줘"
```

> `/전체완성` 하나로 위 4개를 순서대로 한 번에 실행할 수도 있습니다.

### STEP 4. 한글 변환

```
/한글변환
```

`final/사업계획서.hwpx`가 생성됩니다.
AI가 `project.yaml`에 설정된 서체, 줄간격, 여백을 자동으로 적용하지만,
**반드시 한글 프로그램에서 열어서 아래 항목을 육안으로 확인하세요:**

- 서체와 크기가 규정대로인지
- 줄간격이 규정대로인지
- 페이지 분량이 제한 이내인지
- 표가 깨지지 않았는지

## AI 에이전트 소개

## 폴더 구조

```
zerocrew/
├── project.yaml                    ← 규정: 예산, 포맷, 섹션, 심사 기준
├── plan.md                         ← 아이디어: 아이템 정보, 팀 정보

├── docs/                            ← 공고문, 양식 등 참고 문서

├── versions/                        ← 팀원별 독립 버전
│   ├── A-1/, A-2/, A-3/            ← A팀원의 3개 버전
│   ├── A/drafts/                    ← A팀원이 선택한 베스트
│   ├── B-1/, B-2/, B-3/            ← B팀원의 3개 버전
│   ├── B/drafts/                    ← B팀원이 선택한 베스트
│   └── ...                          ← C, D팀원도 동일
│
├── drafts/                          ← 버전심사관이 선정한 최종 조합
│
├── final/                           ← 최종 산출물
│   ├── 사업계획서.md                 ← 문서통합자가 생성
│   └── 사업계획서.hwpx               ← 한글변환기가 생성 (제출용!)
│
├── research/                        ← 리서치 결과물
│
├── .claude/
│   ├── agents/                      ← AI 에이전트 10종
│   ├── commands/                    ← /슬래시 명령어 13종
│   └── skills/                      ← 도메인 지식 스킬 6종
│
├── CLAUDE.md                        ← AI 설정 파일
└── README.md                        ← 지금 읽고 있는 이 파일
```

## 제약 조건

`/프로젝트설정`을 실행하면 아래 항목이 `project.yaml`에 자동 설정됩니다:

| 항목 | 설명 |
|------|------|

| 제출 서류 | 필요 서류 목록 |

> 공고문 PDF/HWP를 `docs/` 폴더에 넣으면 AI가 자동으로 규정을 추출합니다.

## 자주 묻는 질문

### Q. Claude Code가 뭔가요?
터미널(명령줄)에서 동작하는 AI 코딩 도우미입니다. 이 프로젝트에서는 사업계획서 작성을 도와주는 AI 팀으로 활용합니다. 코딩 경험이 없어도 사용할 수 있습니다.

### Q. 터미널은 어떻게 여나요?
- **Mac**: `Cmd + Space` → "터미널" 검색 → 엔터
- **Windows**: `Win + R` → `cmd` 입력 → 엔터 (또는 Windows Terminal 사용)
- **Linux**: `Ctrl + Alt + T`

### Q. 비판리뷰가 너무 혹독한데요?
의도된 것입니다. 악마의변호인은 **일부러** 최대한 비판적으로 리뷰합니다. 심사위원이 던질 질문을 미리 찾아내서, 제출 전에 약점을 보강하는 것이 목적입니다. 상처받지 마세요!

### Q. Git 사용법을 모르겠어요
이 프로젝트에서 알아야 할 Git 명령어는 딱 3개입니다:
```bash
git pull                    # 다른 팀원의 최신 변경사항 가져오기
git add . && git commit -m "내 초안 작성"   # 내 작업 저장
git push                    # 팀원에게 공유
```
