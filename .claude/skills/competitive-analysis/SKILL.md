# 경쟁사 분석 프레임워크

## 경쟁 유형 분류

### 직접 경쟁 (Direct)
같은 문제를 같은 방식으로 해결하는 서비스.
- 예: 배달의민족 vs 쿠팡이츠 (둘 다 음식 배달 앱)

### 간접 경쟁 (Indirect)
같은 문제를 다른 방식으로 해결하거나, 다른 문제를 같은 방식으로 해결.
- 예: 배달의민족 vs 편의점 도시락 (같은 니즈, 다른 방식)

### 대안재 (Alternatives)
현재 사용자들이 우리 서비스 없이 문제를 해결하는 방법.
- 예: 엑셀, 카톡 단톡방, 수작업, 종이

> "경쟁사가 없다"는 결론은 불가능합니다. 반드시 대안재라도 분석하세요.

## 포지셔닝 맵 설계

### 축 선정 가이드 (의미 있는 2축)
좋은 축의 조건:
1. 사용자가 실제로 비교하는 기준
2. 경쟁사 간 차이가 명확한 기준
3. 우리가 유리한 위치를 잡을 수 있는 기준

### 축 예시
| X축 후보 | Y축 후보 |
|----------|----------|
| 가격 (저가↔고가) | 기능 수 (단순↔다기능) |
| 편의성 (복잡↔간편) | 전문성 (일반↔전문) |
| 타겟 (개인↔기업) | 커스터마이징 (표준↔맞춤) |

### 표현 방식 (텍스트)
```
         기능 많음
            ↑
    경쟁B   |   경쟁A
            |
 저가 ------+------ 고가
            |
     ★우리  |
            ↓
         기능 적음
```

## 차별화 포인트 도출

### 차별화 유형
1. **기능 차별화**: 경쟁사에 없는 핵심 기능
2. **UX 차별화**: 같은 기능이지만 더 편한 사용 경험
3. **타겟 차별화**: 경쟁사가 무시하는 세그먼트에 집중
4. **가격 차별화**: 무료/저가 전략 또는 프리미엄

### 차별화 문장 공식
❌ "우리는 ○○가 없는 시장의 공백을 채운다" (경쟁 부재 주장)
✅ "경쟁사 A가 [기능X]에 집중하는 반면, 우리는 [기능Y]를 통해 [타겟Z]에게 [가치W]를 제공한다"

## 경쟁사 조사 방법
1. 구글/네이버 검색: "[카테고리] 앱/서비스"
2. 앱스토어/플레이스토어: 카테고리 검색 + 리뷰 분석
3. 크런치베이스/로켓펀치: 스타트업 정보
4. 블로그/기사: "○○ 비교", "○○ 대안"
5. 사용자 커뮤니티: 에브리타임, 블라인드 등
