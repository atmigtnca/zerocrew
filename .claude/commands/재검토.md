사업계획서 전체를 품질검토관 + 악마의변호인으로 재검토해줘.

대상: $ARGUMENTS (없으면 final/사업계획서.md)

## 프로세스

### 1. 품질검토관 실행
- project.yaml 기반 필수항목 체크
- 섹션 간 교차 검증 (수치, 용어, 팀원 수)
- 분량 적정성 확인
- 문체 검토 (3인칭 시점 잔존 여부, 격식체 통일)

### 2. 악마의변호인 실행
- 7개 공격 벡터로 약점 공격
- 심사위원 예상 질문 Top 5

### 3. 결과 병합
두 에이전트의 결과를 우선순위별로 통합:
```
🔴 Critical (품질검토관) + 💀 치명적 (악마의변호인) → 반드시 수정
🟡 Important + 🔥 심각 → 권장 수정
🟢 Nice-to-have + ⚡ 경고 → 선택적
```
