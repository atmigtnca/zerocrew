## 문체 규칙 (사업계획서 공문서체)

아래 문체 규칙을 반드시 준수하여 작성해줘.

### 시점: 1인칭 팀 시점
- 주어: "본 팀", "[팀명]", "저희" (공문서용)
- 금지: "이 팀은", "해당 서비스는", "그들은" (제3자 관찰 시점)
- 우리가 직접 쓴 글이어야 함. 컨설턴트 보고서가 아님.

### 톤: 공문서 격식체
- 종결어미: ~합니다/~입니다 (격식체)
- 금지: ~임/~함 (보고체), ~해요 (구어체), ~이다 (논문체)
- 자신감 있되 과장 없이. 겸손하되 비굴하지 않게.

### 좋은 예 vs 나쁜 예
❌ "이 서비스는 대학생들의 시간 관리 문제를 해결하는 플랫폼이다."
   → 3인칭 + 논문체. 누가 쓴 건지 모름.

✅ "본 팀은 대학생의 시간 관리 문제를 해결하기 위해 ○○ 플랫폼을 개발합니다."
   → 1인칭 팀 + 격식체. 우리가 직접 쓴 계획서.

❌ "시장 규모는 약 5조원으로 추정되며, 성장 가능성이 높은 분야이다."
   → 보고서 스타일. 관찰자 시점.

✅ "본 팀이 진입하고자 하는 ○○ 시장은 약 5조원 규모이며, 연평균 12% 성장하고 있습니다."
   → 우리의 의지가 담긴 문장.

❌ "경쟁사 대비 차별화 포인트는 다음과 같다."
   → 분석 보고서 톤.

✅ "기존 서비스와 달리, 본 팀은 ○○에 집중하여 차별화하겠습니다."
   → 계획과 의지가 드러남.

## 작성 프로세스 (반드시 이 순서대로)

### Phase 1: 초안 생성
각 섹션을 아래 파일로 나눠서 작성:
- 00-아이템개요.md (요약서 1p)
- 01-팀구성.md (0.5p)
- 02-문제인식.md (2.5p) — 시장분석가 에이전트로 시장조사 포함
- 03-실현가능성.md (1.5p) — 전략자문가 에이전트로 경쟁분석 포함
- 04-사업비편성.md (0.5p) — 예산분석가 에이전트로 예산 검증 포함

시장 데이터와 경쟁사 정보는 실제 웹 검색으로 수집해줘.
같은 팀원의 이전 버전(같은 알파벳의 다른 번호)이 있으면 읽어보고, 다른 관점/내러티브로 작성해줘.

### Phase 2: 악마의변호인 공격
초안 작성이 끝나면, 악마의변호인 서브에이전트를 호출하여 방금 작성한 초안을 비판적으로 리뷰해줘.
7개 공격 벡터(논리, 근거, 시장, 경쟁, 실행, 재무, 치명적 질문)로 약점을 찾아내줘.

### Phase 3: 보강
악마의변호인이 찾아낸 약점을 바탕으로 초안을 수정해줘:
- 💀 치명적 결함: 반드시 수정
- 🔥 심각한 약점: 가능한 수정
- ⚡ 경고: 시간이 되면 수정

수정 후 최종 파일을 같은 폴더에 덮어써줘.

### Phase 4: 결과 보고
사용자에게 아래 내용을 보고:
1. 생성된 파일 목록
2. 악마의변호인이 찾은 주요 약점 3개
3. 보강한 내용 요약
4. 남아있는 약점 (있으면)
