## Writing Style Rules (business plan formal style)

Follow these style rules strictly.

### Perspective: first-person team
- Subject: "본 팀", "[team name]", "저희" (formal)
- Prohibited: "이 팀은", "해당 서비스는", "그들은" (third-person observer)
- Must read as written by the team. Not a consultant report.

### Tone: formal polite (공문서 격식체)
- Sentence endings: ~합니다/~입니다 (formal polite) only
- Prohibited: ~임/~함 (report), ~해요 (casual), ~이다 (academic)
- Confident without exaggeration. Humble without being servile.

### Good vs Bad examples
[BAD] "이 서비스는 대학생들의 시간 관리 문제를 해결하는 플랫폼이다."
   -> Third-person + academic style. Unknown author.

[GOOD] "본 팀은 대학생의 시간 관리 문제를 해결하기 위해 ○○ 플랫폼을 개발합니다."
   -> First-person team + formal polite. Written by us.

[BAD] "시장 규모는 약 5조원으로 추정되며, 성장 가능성이 높은 분야이다."
   -> Report style. Observer perspective.

[GOOD] "본 팀이 진입하고자 하는 ○○ 시장은 약 5조원 규모이며, 연평균 12% 성장하고 있습니다."
   -> Our intent is clear.

[BAD] "경쟁사 대비 차별화 포인트는 다음과 같다."
   -> Analysis report tone.

[GOOD] "기존 서비스와 달리, 본 팀은 ○○에 집중하여 차별화하겠습니다."
   -> Plan and intent are visible.

## Writing Process (follow this order strictly)

### Phase 1: Draft Generation
Write each section as a separate file:
- 00-아이템개요.md (summary, 1p)
- 01-팀구성.md (0.5p)
- 02-문제인식.md (2.5p) -- use the market analyst agent for market research
- 03-실현가능성.md (1.5p) -- use the strategy advisor agent for competitor analysis
- 04-사업비편성.md (0.5p) -- use the budget analyst agent for budget verification

Collect market data and competitor information through actual web searches.
If previous versions by the same team member (same letter, different number) exist, read them and write from a different perspective/narrative.

### Phase 2: Devil's Advocate Attack
After drafting is complete, invoke the devil's advocate sub-agent to critically review the draft you just wrote.
Find weaknesses using 7 attack vectors (logic, evidence, market, competition, execution, finances, fatal questions).

### Phase 3: Reinforcement
Revise the draft based on the weaknesses found by the devil's advocate:
- Fatal flaws: must fix
- Serious weaknesses: fix if possible
- Warnings: fix if time allows

After revision, overwrite the final files in the same folder.

### Phase 4: Results Report
Report the following to the user:
1. List of generated files
2. Top 3 weaknesses found by the devil's advocate
3. Summary of reinforcements made
4. Remaining weaknesses (if any)
