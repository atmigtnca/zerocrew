plan.md를 읽고, 나의 버전 폴더에 전체 사업계획서(섹션 0~4)를 작성해줘.

작성할 폴더: versions/$ARGUMENTS/drafts/

예시: `/초안작성 A-1` → versions/A-1/drafts/ 에 생성
예시: `/초안작성 B-2` → versions/B-2/drafts/ 에 생성

## 작성 프로세스 (반드시 이 순서대로)

### Phase 1: 초안 생성
각 섹션을 아래 파일로 나눠서 작성:
- 00-아이템개요.md (요약서 1p)
- 01-팀구성.md (0.5p)
- 02-문제인식.md (2.5p) — 시장분석가 에이전트로 시장조사 포함
- 03-실현가능성.md (1.5p) — 전략자문가 에이전트로 경쟁분석 포함
- 04-사업비편성.md (0.5p) — 예산분석가 에이전트로 예산 검증 포함

시장 데이터와 경쟁사 정보는 실제 웹 검색으로 수집해줘.
같은 팀원의 이전 버전(같은 알파벳의 다른 번호)이 있으면 읽어보고, 다른 관점/내러티브로 작성해줘.

### Phase 2: 악마의변호인 공격
초안 작성이 끝나면, 악마의변호인 서브에이전트를 호출하여 방금 작성한 초안을 비판적으로 리뷰해줘.
7개 공격 벡터(논리, 근거, 시장, 경쟁, 실행, 재무, 치명적 질문)로 약점을 찾아내줘.

### Phase 3: 보강
악마의변호인이 찾아낸 약점을 바탕으로 초안을 수정해줘:
- 💀 치명적 결함: 반드시 수정
- 🔥 심각한 약점: 가능한 수정
- ⚡ 경고: 시간이 되면 수정

수정 후 최종 파일을 같은 폴더에 덮어써줘.

### Phase 4: 결과 보고
사용자에게 아래 내용을 보고:
1. 생성된 파일 목록
2. 악마의변호인이 찾은 주요 약점 3개
3. 보강한 내용 요약
4. 남아있는 약점 (있으면)
