사업계획서를 비판 → 선택 → 수정 루프로 고도화해줘.

대상: $ARGUMENTS (없으면 final/사업계획서.md)

## 프로세스

### Round 1
1. 악마의변호인 서브에이전트(최종 모드)로 현재 초안 공격
2. 약점을 번호로 정리하여 표시:
   ```
   💀 1. [치명적 결함 제목] — [파일:위치]
   🔥 2. [심각한 약점 제목] — [파일:위치]
   🔥 3. [심각한 약점 제목] — [파일:위치]
   ⚡ 4. [경고 제목]
   ```
3. 사용자에게 질문: "어떤 항목을 수정할까요? (번호, 쉼표로 구분 또는 직접 지시)"

### 수정
4. 선택된 약점을 자동 수정 (문체 규칙 준수: 1인칭 팀 시점, 격식체)
5. 수정 내용 요약 표시

### Round 2 (자동)
6. 수정된 초안을 악마의변호인이 재공격
7. 결과 표시: "💀 N건, 🔥 M건 남음"
8. 💀이 0건이면 자동 종료
9. 💀이 남아있으면: "계속할까요?" (최대 3라운드)

### 종료
10. 최종 상태 요약: 수정 전/후 비교, 남은 약점 수
